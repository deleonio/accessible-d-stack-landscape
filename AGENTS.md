# Agent Instructions

This repository is a Preact + Vite application that visualizes technologies, components, and protocols for an accessible digital platform landscape. It uses KoliBri web components, UnoCSS, and SCSS for styling.

## Setup Commands

```bash
# Install dependencies (also runs postinstall: asset copy + article generation)
pnpm i

# Start development server (http://localhost:5173)
pnpm dev

# Build for production
pnpm build
```

## Code Quality

```bash
# Run all linters (TypeScript, ESLint, Stylelint) in parallel (via npm-run-all2)
pnpm lint

# Individual linters:
pnpm lint:ts        # TypeScript type check
pnpm lint:eslint    # ESLint
pnpm lint:stylelint # Stylelint (SCSS)

# Check formatting
pnpm format

# Unused exports/dependencies
pnpm check-unused
```

## Testing

```bash
# Run Playwright E2E tests
pnpm test:e2e

# Interactive test UI
pnpm test:e2e:ui

# Update visual snapshots
pnpm test:e2e:update-snapshots
```

## Project Structure

- `src/` ‚Äì Application source code
  - `components/` ‚Äì Preact UI components (TSX)
  - `data/` ‚Äì Generated and static data files
  - `hooks/` ‚Äì Custom Preact hooks
  - `types/` ‚Äì TypeScript type definitions
  - `utils/` ‚Äì Utility functions
  - `App.tsx` ‚Äì Main application component
  - `style.scss` ‚Äì Global styles and design tokens
- `scripts/` ‚Äì Build-time generators (article data)

- `index.html` ‚Äì Entry HTML with inline splash animation
- `.github/workflows/` ‚Äì CI, deploy, and cleanup workflows

## Tech Stack

- **UI:** Preact 10 with JSX (`jsxImportSource: preact`)
- **Build:** Vite
- **Styling:** UnoCSS (presetMini) + SCSS
- **Components:** KoliBri (`@public-ui/components` + `@public-ui/preact`)
- **Forms:** React Hook Form + Zod validation
- **PWA:** vite-plugin-pwa (toggleable via `VITE_ENABLE_PWA` env)
- **Mobile:** Capacitor (Android/iOS)
- **E2E Tests:** Playwright + axe-core
- **Linting:** ESLint (with `jsx-a11y`, `react-hooks`), Stylelint, Prettier, Knip

## üö® KoliBri First ‚Äì Non-Negotiable Rule

**ALWAYS use KoliBri components (`@public-ui/preact`) for EVERYTHING that KoliBri provides.**
Using native HTML elements or custom implementations when a KoliBri equivalent exists is FORBIDDEN.

### Which components are available?

**Use the MCP tool `mcp_kolibri-mcp_search` or `mcp_kolibri-mcp_fetch` BEFORE implementing any UI element** to check if KoliBri has a component for it. Do NOT guess from memory.

Examples of what KoliBri covers (non-exhaustive):

- Buttons ‚Üí `KolButton`
- Checkboxes / Switches ‚Üí `KolInputCheckbox _variant="switch"`
- Select dropdowns ‚Üí `KolSingleSelect`
- Text inputs ‚Üí `KolInputText`
- Pagination ‚Üí `KolPagination`
- Cards ‚Üí `KolCard`
- Drawers / Side panels ‚Üí `KolDrawer`
- Images ‚Üí `KolImage`
- Badges / Tags ‚Üí use KoliBri where possible
- Links ‚Üí `KolLink`
- Tabs ‚Üí `KolTabs`
- Accordion ‚Üí `KolAccordion`
- Alerts / Notifications ‚Üí `KolAlert`
- Modals ‚Üí `KolModal`
- Progress ‚Üí `KolProgress`
- Tooltips ‚Üí `KolTooltip`
- Spinners ‚Üí `KolSpin`

### MCP usage is MANDATORY

Before building any UI component or pattern, run:

```
mcp_kolibri-mcp_search(query: "<component name or use case>")
```

If a KoliBri component exists ‚Üí use it. If uncertain ‚Üí fetch the docs:

```
mcp_kolibri-mcp_fetch(url: "https://kolibri.digital/<component>")
```

Only fall back to custom HTML/CSS if KoliBri provably has no solution.

## Coding Conventions

- Use ESM imports exclusively (`"type": "module"`).
- TypeScript strict mode is enabled.
- JSX uses Preact's `jsxImportSource`, not React.
- Do not manually edit `public/assets/` ‚Äì these are copied from KoliBri packages during `postinstall`.
- Do not manually edit `src/data/articles.generated.ts` ‚Äì it is generated by `scripts/generate-articles.mjs` using `logo-urls.json` for fallback resolution.
- All article logos are guaranteed to display: external URL, local asset, or `/assets/broken-logo.svg` (for items without public logo).
- Use exact version numbers in `package.json` dependencies.
- Keep lists and enumerations in alphabetical order.
- Formatting is enforced via Prettier (tabs, single quotes).

## Deployment

- **main branch:** Auto-deploys to GitHub Pages root via `deploy-main.yml`. PR preview folders (`pr-*`) are preserved during deployment.
- **Pull requests:** Preview deployments to `pr-<number>/` via `deploy-pr.yml`.
- **Cleanup:** `cleanup-pr.yml` removes PR previews on PR close. `cleanup-gh-pages.yml` removes orphaned previews.
